# REGLAS CRÍTICAS DE DESARROLLO

## 1. SEGURIDAD PRIMERO
- **PROHIBIDO** modificar código existente sin antes explicar el cambio en el chat.
- **SIEMPRE** espera mi confirmación ("OK", "Dale", "Procede", "Si") antes de escribir en archivos.
- Si tienes dudas sobre una función, **PREGUNTA**, no supongas.

## 2. USO DE SKILLS
- Consulta el archivo `SKILL.md` dentro de la carpeta `.agents/skills/` que corresponda a la tarea (ej. `interface-design` para UI).
- Ejecuta las habilidades de forma secuencial, no saltes pasos.

## 3. ESTILO DE CÓDIGO
- Mantén el naming convention (nombres de variables/clases) y la estructura de carpetas actual.
- No añadas librerías nuevas sin permiso.
- Haz cambios pequeños y específicos (atómicos).

## 4. FLUJO DE TRABAJO
1. Analiza la tarea y el archivo solicitado.
2. Propón el cambio en el chat.
3. Tras mi aprobación, aplica el código con comentarios claros.
4. Confirma cuando hayas terminado.
